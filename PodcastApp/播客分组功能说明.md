# 播客分组功能说明

## 功能概述

为播客添加了分组/标签功能，用于区分不同来源和类型的播客。

## 实现内容

### 1. 数据模型更新

在 `Podcast` 模型中新增 `category` 字段：
- **类型**：String
- **用途**：标识播客的分组/来源
- **默认值**："系统推荐"

### 2. 分组规则

#### 系统推荐
- **来源**：用户定时生成（早上7点等固定时间）
- **特点**：包含所有话题的综合内容
- **标签**："系统推荐"

#### 话题播客
- **来源**：话题自动生成（每2小时等间隔）
- **特点**：单个话题的专属内容
- **标签**：话题名称（如"Swift 开发"、"AI 技术"等）

### 3. 界面展示

#### 播客列表
- **分组显示**：按 category 分组展示播客
- **分组头部**：显示分组名称和播客数量
- **固定头部**：滚动时分组头部保持可见

#### 播客卡片
- **标签显示**：在标题右侧显示分组标签
- **样式**：浅色背景 + 主题色文字
- **位置**：标题行右侧

### 4. 生成逻辑

#### 用户定时生成
```swift
podcastService.generatePodcast(
    topics: allTopics,
    config: config,
    modelContext: modelContext,
    category: "系统推荐"
)
```

#### 话题自动生成
```swift
podcastService.generatePodcast(
    topics: [singleTopic],
    config: config,
    modelContext: modelContext,
    category: topic.name
)
```

## 使用场景

### 场景1：综合浏览
用户想了解所有话题的最新动态：
- 查看"系统推荐"分组
- 包含所有关注话题的精选内容

### 场景2：专题深入
用户想深入了解某个特定话题：
- 查看对应话题分组（如"Swift 开发"）
- 只包含该话题的相关内容

### 场景3：内容管理
用户可以清晰地看到：
- 哪些是系统综合推荐的内容
- 哪些是特定话题的专属内容
- 每个分组有多少播客

## 数据迁移

对于已有的播客数据：
- 如果没有 `category` 字段，默认为"系统推荐"
- SwiftData 会自动处理模型迁移
