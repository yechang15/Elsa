# UI 设计文档：对话式播客应用 V0.1

> macOS 原生桌面应用，基于 Swift + SwiftUI

---

## 一、整体架构

### 1.1 应用结构

```
┌─────────────────────────────────────────────────────────────────┐
│  标题栏：应用名称 + 窗口控制                                      │
├──────────────┬──────────────────────────────────────────────────┤
│              │                                                    │
│   侧边栏     │              主内容区                              │
│   (200px)    │              (可变宽度)                            │
│              │                                                    │
│              │                                                    │
│              │                                                    │
│              │                                                    │
│              │                                                    │
├──────────────┴──────────────────────────────────────────────────┤
│  底部播放控制栏 (固定高度 80px)                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 侧边栏导航

侧边栏包含以下导航项：
- 🎧 播客列表（默认选中）
- 🎯 兴趣话题
- 📡 RSS 订阅
- 📊 收听历史
- ⚙️ 设置

**交互说明**：
1. LLM 配置：选择 API 提供商、输入 API Key、选择模型
2. TTS 配置：选择引擎（系统 TTS/OpenAI TTS/ElevenLabs）、调整语速、选择音色
3. 播客生成：设置默认长度、内容深度、主播风格、自动生成时间
4. 通知：勾选需要接收的通知类型
5. 外观：选择主题（浅色/深色/跟随系统）
6. 所有设置实时保存

## 五、交互流程

### 5.1 首次启动流程

```
启动应用
  ↓
显示话题选择界面
  ↓
用户选择话题（至少 1 个）
  ↓
点击"开始使用"
  ↓
系统自动订阅 RSS 源
  ↓
进入主界面（播客列表）
  ↓
后台开始生成第一批播客
```

### 5.2 播客生成与播放流程

```
系统周期性拉取 RSS 内容
  ↓
根据话题和内容生成播客脚本
  ↓
TTS 转换为音频
  ↓
播客出现在列表中
  ↓
用户点击播客
  ↓
进入播放界面
  ↓
开始播放
  ↓
记录收听历史和进度
```

### 5.3 话题管理流程

```
用户进入兴趣话题页面
  ↓
点击"添加话题"
  ↓
从推荐列表选择或自定义输入
  ↓
系统自动映射相关 RSS 源
  ↓
开始订阅并生成播客
```

## 六、响应式设计

### 6.1 窗口尺寸

- **最小窗口尺寸**：1000px × 600px

1. 窗口宽度 < 1000px 时，侧边栏自动折叠为图标模式
2. 播客卡片根据窗口宽度自动调整布局
3. 播放控制栏元素在窄窗口时优先显示核心控制
